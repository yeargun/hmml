the medium

Pixels are an output. Not a source.

Image formats compress the result of drawing. HMML stores the drawing itself — declarative space, motion and logic — and stays smaller doing it.

Compress intent

Don't freeze the frame. Keep the program that paints it: layout, 3D, animation, behaviour.

If a browser can render it

…HMML can hold it. HTML, CSS, SVG, canvas, JS — one binary document, no folder of assets.

Generative by design

Built for generative media: declare a scene, a loop, a system — not a grid of dead pixels.

the contract

bits & bytes & a contract.

A PNG-style signature, then self-describing chunks. Markup stays text; images stay raw. Unknown chunks are skipped, so it grows without breaking. Full spec →

89 'H' 'M' 'M' 'L' 0D 0A 1A 0A  ·  major · minor · codec
MARK
the markup — HTML/CSS/SVG/JS, gzip-able text
RSRC
a resource — id + mime + raw bytes (no base64)
META

ENDF

use it

Two calls.

// pack any HTML (data: images are pulled out into raw bytes, gzip by default)
const bytes = await pack(`<section style="transform:matrix3d(…)"><img src="data:image/webp;base64,…"></section>`)

// …store / send the .hmml … then anywhere a browser runs:
const doc = await unpack(bytes)
el.innerHTML = doc.toHTML()
